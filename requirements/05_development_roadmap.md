# 開発ロードマップ (DEVELOPMENT_ROADMAP)

## 1. 目的
本ドキュメントは、プロジェクトを現実的かつ管理可能なステップに分割し、段階的な機能リリースを計画するために使用する。

## 2. 全体方針
- 最初は、サーバーを持たない完全にクライアントサイドで完結するMVP（Minimum Viable Product）を開発し、コアとなる学習体験を確立する。
- 次に、サーバーサイドの機能を導入し、ユーザー間のデータ共有やコミュニティ機能を実装する。
- 最終的に、プラットフォームとしての価値を高め、収益化を目指す。

## 3. フェーズ別実装計画

### フェーズ1: コア体験の完成 (ローカル完結型MVP)
- **目標:** 最高の「個人用」記憶支援ツールを迅速に開発し、リリースする。
- **インフラ/データ:** サーバーなし。データは全てブラウザ内（Local Storage / IndexedDB）に保存。
- **実装する機能:**
  - `[機能要件 1.1]` **知識カード登録機能**
    - テキストおよび画像の登録
  - `[機能要件 1.2]` **復習機能**
    - 「今日のデッキ」の自動生成
    - 忘却曲線に基づく学習スケジュールの更新
  - `[機能要件 1.3]` **Webクリッパー機能 (Chrome拡張)**
    - テキスト・画像の選択からのカード作成
  - `[機能要件 1.4]` **学習ダッシュボード機能**
    - 総カード数、定着度、連続学習日数の表示

### フェーズ2: コミュニティ機能の追加 (サーバー導入)
- **目標:** ユーザー間で学習コンテンツを共有できるプラットフォームを構築する。
- **インフラ/データ:** サーバーサイド（Supabase, Next.js API Routes）を導入。データはクラウドDBに永続化。
- **実装する機能:**
  - **ユーザーアカウント機能**
    - Supabase Authを利用したメールアドレス/パスワードおよびソーシャルログイン
  - `[機能要件 1.5]` **デッキ共有・インポート機能**
    - 作成したデッキのサーバーへのアップロード（公開/非公開設定）
    - 他ユーザーが公開したデッキの検索およびインポート
  - **マルチデバイス同期**
    - クラウドに保存されたデータに、異なるデバイスから同じアカウントでアクセスできる。

### フェーズ3: マーケットプレイス化 (収益化)
- **目標:** コンテンツクリエイターが収益を得られるエコシステムを構築し、ビジネスとして持続可能なモデルを確立する。
- **インフラ/データ:** 決済システム（Stripeなど）を導入。
- **実装する機能:**
  - `[機能要件 1.6]` **デッキマーケットプレイス機能**
    - デッキへの価格設定機能
    - クレジットカード等によるオンライン決済機能
    - 販売手数料の徴収システム
    - クリエイターへの収益分配システム
  - **評価・レビュー機能**
    - 公開/販売されているデッキに対するユーザーレビューや5段階評価
  - **高度なデッキ検索機能**
    - 人気ランキング、評価順、新着順などでのデッキ検索
