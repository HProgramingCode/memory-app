# 現状の課題

## ページ挙動
1. ページリロード時に初期HTMLが表示される
2. クライアントで Zustand / Session / Theme が初期化され、UIが一度書き換わる
3. データ取得後に最終状態に更新される

→ ユーザーには **画面が段階的に切り替わる（チラつく / 違う画面が見える）** ように見える

## 原因
1. **Server と Client の初期状態が一致していない**  
   - Zustand の初期 state が空 → `useEffect` で後から置き換え
2. **DataInitializer が Client 側で状態を書き換えている**  
   - 再描画が必ず発生
3. **SessionProvider の非同期初期化**  
   - ログイン状態確定前に UI が描画される
4. **ThemeRegistry / Emotion の SSR CSS 遅延**  
   - CSSやフォントがクライアントで注入されるため、一瞬素のレイアウトになる

## 影響
- UIが一瞬崩れる / 違う画面が表示される  
- ユーザー体験が乱れる  
- 初回表示で「画面が3段階で切り替わる」ように見える

## 課題の本質
> **サーバーで返すHTMLとクライアントで初期化される状態の差がユーザーに見えてしまっている**

### 要点
- **UIの途中状態をユーザーに見せずに、最終形で初回レンダリングしたい**
