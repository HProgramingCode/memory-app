# フロントエンド美学方向性 (FRONTEND_AESTHETIC_DIRECTION)

## 1. インターフェースの目的

学習者が「知識の登録」と「復習」に集中できる、ミニマルで心地よい学習環境を提供する。  
視覚的なノイズを最小化し、学習のリズム感と達成感を演出する。

## 2. トーン / スタイル

**Clean Minimal + Warm Functional**

- ミニマリズムをベースに、冷たくなりすぎない温かみのあるデザイン
- 学習ツールとして「使いやすさ」と「居心地の良さ」を両立
- Anki 等の既存 SRS ツールが持つ「機能的だが無機質」な印象を払拭する
- Material Design の構造・一貫性を活かしつつ、独自の柔らかさを加える

## 3. カラーパレット

### プライマリ

- **Primary:** `#5C6BC0`（Indigo 400 — 知的で落ち着いた印象）
- **Primary Light:** `#8E99A4`
- **Primary Dark:** `#3949AB`

### アクセント

- **Success / 簡単:** `#66BB6A`（Green — ポジティブな達成感）
- **Warning / 普通:** `#FFA726`（Orange — 注意・中間）
- **Error / 難しい:** `#EF5350`（Red — 注意喚起）

### 背景・テキスト

- **Background:** `#FAFAFA`（限りなく白に近いグレー）
- **Surface:** `#FFFFFF`
- **Text Primary:** `#212121`
- **Text Secondary:** `#757575`

### ダークモード

- Phase 1 では非対応。将来的に MUI の `ThemeProvider` で切り替え可能な設計にしておく。

## 4. タイポグラフィ

- **フォント:** MUI デフォルト（Roboto）。日本語環境では `"Roboto", "Noto Sans JP", sans-serif` のフォールバック
- **見出し:** 太めのウェイト（500-700）で明確な視覚階層
- **本文:** 400 ウェイト、`1.6` の行間で読みやすさを確保
- **カードコンテンツ:** Markdown レンダリング部分は `1.7` の行間で余裕を持たせる

## 5. レイアウト・構図

- **最大幅:** `1200px`（デスクトップ）。コンテンツが画面いっぱいに広がらないようにする
- **余白:** 十分な padding と margin で「呼吸感」を出す
- **カードコンポーネント:** MUI の `Card` に `elevation={1}` 程度の控えめなシャドウ。角丸 `12px`
- **復習画面:** カードを画面中央に大きく配置。distraction-free な環境
- **レスポンシブ:** モバイルではシングルカラム。タブレット以上で 2 カラムレイアウトをダッシュボードに適用

## 6. アニメーション・インタラクション

- **控えめかつ意味のあるモーション** を基本方針とする
- カード表示の切り替え: `fade` トランジション（200-300ms）
- 「答えを見る」アクション: 裏面が `slide-up` でスムーズに表示
- 評価ボタン押下後の次カード遷移: `fade-out → fade-in`
- ダッシュボードの数値: ページロード時に軽いカウントアップアニメーション
- ホバーエフェクト: `elevation` の微妙な変化（1 → 3）

## 7. 差別化要素

- **ストリーク表示:** 炎アイコンとカウント数字で連続学習のモチベーションを視覚的に演出
- **復習画面のフォーカスモード:** ヘッダーを最小化し、カードとアクションボタンのみに集中
- **学習サマリーの円グラフ:** 定着度を直感的に把握できる視覚要素
- **空状態のデザイン:** カードやデッキがない状態でも、やさしいイラストやメッセージで導線を示す
