# プロジェクト定義 (PROJECT_DEFINITION)

## 1. プロジェクト名
- Memory App

## 2. 目的 (Purpose)
- **Primary Goal:** 学習者が登録したあらゆる知識（単語、概念、コードスニペット、画像など）を、効率的に長期記憶へ定着させることを支援する。
- **Core Technology:** 科学的に証明された「忘却曲線」および「間隔反復学習（Spaced Repetition System; SRS）」の理論を中核技術として利用する。

## 3. コアコンセプト (Core Concept)
- 本アプリケーションは、既存のSRSツール（例: Anki）が持つ「高機能だが複雑」という課題を解決し、以下の3つの特徴によって差別化を図る。
  1.  **シンプルさ (Simplicity):** 複雑な設定を排除し、ユーザーが「知識の登録」と「復習」という本質的な行動に集中できる、直感的でミニマルなUI/UXを提供する。
  2.  **入力の効率化 (Effortless Input):** Webクリッパー機能などにより、日常生活や学習の中で得た知識を、可能な限り手間なくシームレスに登録できる仕組みを提供する。
  3.  **アクセシビリティ (Accessibility):** インストール不要のWebアプリケーションとして提供し、あらゆるデバイスのブラウザからアクセス可能にすることで、学習の機会を最大化する。

## 4. ターゲットユーザー (Target Audience)
- **ペルソナ1: 資格試験の受験生**
  - 課題: 膨大な暗記項目を効率的に学習したい。どの項目をいつ復習すれば良いか分からない。
  - 期待: 自分の学習状況に合わせて、復習すべき問題を自動で提示してほしい。
- **ペルソナ2: 語学学習者**
  - 課題: 新しい単語やフレーズがなかなか覚えられない。単語カードを作るのが面倒。
  - 期待: Web上の記事や動画で出会った未知の単語を、その場で簡単に登録して復習したい。
- **ペルソナ3: 新しい技術を学ぶエンジニア・学生**
  - 課題: コードスニペットや設計思想、専門用語などを記憶に定着させたい。
  - 期待: 図やスクリーンショットと共に、技術的な知識を整理・記憶したい。
- **ペルソナ4: 知的好奇心旺盛な読書家**
  - 課題: 本を読んでも内容をすぐに忘れてしまう。「積読」ならぬ「読み忘れ」を減らしたい。
  - 期待: 本で得た重要な気づきや引用を、手軽に登録して血肉としたい。
