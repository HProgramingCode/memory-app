## Purpose
Build frontend components, pages, or applications according to a pre-defined aesthetic direction.
Focus on functional, visually appealing, and polished implementation.

あらかじめ決められた美学の方向性に沿って、フロントエンドのコンポーネント、
ページ、アプリケーションを構築します。機能性と視覚的完成度を両立させることに重点を置きます。

## Input
- Structured requirements from `define-requirements`
- Technology choices from `decide-tech-stack`
- Aesthetic direction brief from `design-frontend-aesthetic-direction` (Markdown)

入力：
- define-requirements から整理済みの要件
- decide-tech-stack からの技術選定情報
- design-frontend-aesthetic-direction からの美学方向性（Markdown）

## Output
- Fully working frontend code (HTML/CSS/JS, React, Vue, etc.)
- Markdown-wrapped code blocks
- Must follow the aesthetic direction
- Should be production-ready, visually distinctive, and cohesive

出力：
- 完全に動作するフロントエンドコード（HTML/CSS/JS, React, Vue 等）
- Markdown コードブロックで出力
- 美学方向性に沿った実装
- 本番品質で、視覚的にも統一感があり特徴的なデザイン

## Execution Guidelines
- Implement exactly according to the aesthetic direction
- Typography, colors, layout, motion, and visual details should reflect the chosen style
- Avoid generic fonts, cliched color schemes, predictable layouts, or cookie-cutter patterns
- Match implementation complexity to the aesthetic vision (maximalist → elaborate, minimalist → precise and refined)
- Keep code readable, maintainable, and testable

実行ガイドライン：
- 美学方向性に正確に従う
- タイポグラフィ、色、レイアウト、アニメーション、背景などを反映
- 汎用フォント、定番色、予測可能なレイアウト、テンプレ的デザインを避ける
- 美学に応じて実装の複雑度を調整（マキシマリスト → 豪華、ミニマリスト → 精密かつ洗練）
- コードは読みやすく、保守可能、テスト可能に保つ
