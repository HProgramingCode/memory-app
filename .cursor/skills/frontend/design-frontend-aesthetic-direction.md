## Purpose
Establish a bold, memorable, and cohesive visual direction for frontend development.
This skill generates a structured aesthetic brief that frontend-design will follow.

フロントエンド開発のために、明確で特徴的な美学方向性を決定します。
生成される美学ブリーフを `frontend-design` が使用して実装します。

## Input
- Structured requirements from `define-requirements`
- Optional context: audience, purpose, brand, references, constraints

入力：
- define-requirements からの整理済み要件
- 任意の文脈情報：対象ユーザー、目的、ブランド、参照例、制約条件

## Output
- Markdown-formatted aesthetic brief including:
  - Purpose / Function of the interface
  - Tone / Style (e.g., minimal, maximalist, retro, organic, luxury, playful, brutalist, geometric, etc.)
  - Typography guidance
  - Color palette and theme
  - Layout and composition notes
  - Motion / interaction guidance
  - Any unique visual differentiators

Markdown形式での美学ブリーフ：
- インターフェースの目的・機能
- トーン / スタイル（ミニマル、マキシマリスト、レトロ、オーガニック、ラグジュアリー、遊び心、ブルータリスト、幾何学的 など）
- タイポグラフィの指針
- カラーパレットとテーマ
- レイアウトや構図の指示
- アニメーション・インタラクション指針
- 独自性や差別化要素

## Execution Guidelines
- Commit to a single cohesive aesthetic direction
- Bold maximalism or refined minimalism is acceptable, but must be intentional
- Ensure visual differentiation and memorability
- Provide concise and actionable guidance for frontend implementation

実行ガイドライン：
- 明確で統一された美学方向性を決定
- マキシマリストでもミニマリストでも、意図的に選択
- 視覚的差別化と記憶に残るデザインを保証
- フロントエンド実装に直接使える簡潔かつ実行可能な指示を提供
